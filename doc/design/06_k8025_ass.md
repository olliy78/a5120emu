# Feinentwurf: K8025 ASS – Anschlußsteuerung Seriell

**Modul:** `core/cards/k8025/`  
**Dateien:** `k8025.h`, `k8025.cpp`  
**Dokumentation:** `doc/trascripted/Anschlußsteuerung K 8025.50 und K 8025.80.md`

---

## 1. Aufgabe

Die K8025 ist die serielle Schnittstellenkarte des A5120. Sie verbindet:
- Tastatur K7637 (IFSS, 9600 Baud, intern)
- Drucker (IFSS, 9600 Baud)
- DFÜ / Modem (V.24 und/oder IFSS, 50–9600 Baud, programmierbar)

---

## 2. Hardware-Zusammensetzung

```
K8025-Karte (Variante .50: V.24 + IFSS)
├── SIO A33 (Q 304) – DFÜ-SIO, Ports 50H–53H
│   ├── Kanal A: DFÜ (V.24/IFSS, X6/X5)
│   └── Kanal B: (nicht belegt / Zusatz)
├── SIO A32 (Q 304) – Drucker/Tastatur-SIO, Ports 5CH–5FH
│   ├── Kanal A: Tastatur K7637 (IFSS, X4)  ← bestätigt
│   └── Kanal B: Hauptdrucker (IFSS, X3)
├── CTC A34 (Q 302) – Taktversorgung, Ports 58H–5BH
│   ├── Kanal 0: Baud-Rate Drucker (fest 9600 Baud)
│   ├── Kanal 1: Baud-Rate DFÜ (programmierbar)
│   └── Kanal 2-3: weitere Takte
└── PIO A31 (Register A31) – Übertragungsparameter, Ports 54H–57H
    └── DIL-Schalter-Abbild (Baudrate, Format für DFÜ)
```

## 4. Klassen-Interface

```cpp
class K8025 : public BusDevice, public InterruptSlave {
public:
    struct A5120Config {
        uint8_t io_base = 0x50;       // Standard: 50H
        bool    has_v24 = true;       // V.24-Schnittstelle vorhanden
        bool    has_ifss_dfue = true; // IFSS für DFÜ vorhanden
    };

    K8025(K1520Bus& bus, Koppelbus& koppel, const A5120Config& cfg = {});

    // BusDevice: Ports 50H–5FH (16 Ports)
    uint8_t ioRead(uint8_t port) override;
    void    ioWrite(uint8_t port, uint8_t data) override;

    // InterruptSlave (Daisy-Chain)
    void    setIEI(bool) override;
    bool    getIEO() const override;
    bool    hasInterrupt() const override;
    uint8_t getVector() const override;

    // Zweite Interrupt-Kette (Koppelbus)
    void    setIEI1(bool);
    bool    getIEO1() const;

    // ─── Serielle Schnittstellen-Zugänge ─────────────────────────

    // Tastatur-IFSS (SIO A32, Kanal A, X4 – bestätigt)
    // Sendet empfangenes Byte an SIO → löst Interrupt aus
    void    keyboardRxByte(uint8_t byte);
    // Gibt Byte zurück das an Tastatur gesendet wird (für LED-Steuerung)
    bool    keyboardTxAvailable() const;
    uint8_t keyboardTxGet();

    // DFÜ-Schnittstelle (SIO A33, Kanal A)
    // Für Python-seitige DFÜ-Emulation (Drucker, Terminal)
    using SerialCallback = std::function<void(uint8_t)>;
    void    setDFUERxCallback(SerialCallback);   // Zeichen empfangen
    void    dfueRxByte(uint8_t byte);            // Byte injizieren
    bool    dfueTxAvailable() const;
    uint8_t dfueTxGet();

    // Drucker-Schnittstelle (SIO A32)
    void    setPrinterRxCallback(SerialCallback);
    bool    printerTxAvailable() const;
    uint8_t printerTxGet();

    // Takt-Eingang (von Koppelbus, falls ZRE-CTC → K8025-CTC)
    void    baudClk(bool rising_edge);

private:
    A5120Config cfg_;
    Z80SIO      sio_dfue_{"K8025-SIO-A33"};     // Ports 50H–53H (DFÜ)
    Z80SIO      sio_kbd_printer_{"K8025-SIO-A32"}; // Ports 5CH–5FH (Kbd+Drucker)
    Z80CTC      ctc_{"K8025-CTC-A34"};          // Ports 58H–5BH
    // Register A31: PIO für DIL-Schalter
    Z80PIO      pio_a31_{"K8025-PIO-A31"};      // Ports 54H–57H

    SerialCallback dfue_rx_cb_;
    SerialCallback printer_rx_cb_;
};
```

## 5. Tastatur-Anbindung (K7637 → K8025 → K1520Bus)

### 5.1 Tastatur-Anschluss: SIO A32 Kanal A (X4)

Die Tastatur K7637 ist an Stecker **X4** angeschlossen, das ist **SIO A32, Kanal A** (Ports 5CH/5DH).

- SIO A32 Kanal A (5CH/5DH): Tastatur K7637 (IFSS, X4)
- SIO A32 Kanal B (5EH/5FH): Hauptdrucker (IFSS, X3)

### 5.2 Datenfluss

```
K7637::sendKeycode(0x41)          // 'A' getippt
    → IFSS-Bit-Stream (9600 Baud, 1+8+1)
    → K8025::keyboardRxByte(0x41) // direkte Injektion im Emulator
    → sio_kbd_printer_.channelA().rxByte(0x41)
    → SIO setzt RxRDY, löst Interrupt aus
    → Z80CPU: Interrupt-Handler bei SIO-Vektor
    → OS liest Port 5CH (SIO A32 Ch A Data) → 0x41
```

---

## 6. Baud-Rate-Konfiguration

Der CTC A34 generiert die Baud-Raten für beide SIO-Chips. Die ZC/TO-Ausgänge des CTC werden als Clock-Eingänge der SIO-Kanäle verwendet.

**Standard-Konfiguration A5120:**
- CTC Kanal 0 → Drucker-SIO (fest 9600 Baud)
- CTC Kanal 1 → DFÜ-SIO (programmierbar: 50–9600 Baud via DIL A41)

**DIL-Schalter A41 Emulation:**
Die Baud-Rate-Einstellung des DIL-Schalters wird als statische Konfiguration im Emulator gesetzt:

```cpp
struct A5120Config {
    uint32_t dfue_baud = 9600;    // Entspricht DIL-Schalter A41
    // weitere DIL-Parameter...
};
```

## 7. Interrupt-Verkettung

Die K8025 enthält drei interrupt-fähige Chips (SIO A33, SIO A32, CTC A34). Die interne Interrupt-Kette auf der K8025-Karte:

```
/IEI → SIO A33 → SIO A32 → CTC A34 → /IEO
```

Die K8025 als Ganzes ist Glied in der K1520-Bus-Interrupt-Kette.

## 8. V.24-Schnittstelle

Die V.24-Schnittstelle (Stecker X6) wird über Callback nach Python geführt:

```c
// C-API
k1520_serial_set_rx_callback(h, K1520_SERIAL_DFU, my_callback, ctx);
k1520_serial_tx(h, K1520_SERIAL_DFU, 0x0D);  // CR an DFÜ senden
```

## 9. Testbarkeit

```python
# tests/python/test_k8025.py
def test_keyboard_rx_triggers_interrupt(k8025_lib, k1520_bus):
    card = k8025_lib.k8025_create(k1520_bus)
    k8025_lib.k8025_keyboard_rx(card, 0x41)  # 'A' injizieren
    # SIO A33 Ch B müsste RxRDY setzen
    assert k8025_lib.k8025_has_interrupt(card)
    # Interrupt-Vektor lesen
    vec = k8025_lib.k8025_int_ack(card)
    assert vec != 0xFF

def test_printer_tx(k8025_lib, k1520_bus):
    received = []
    card = k8025_lib.k8025_create(k1520_bus)
    k8025_lib.k8025_set_printer_callback(card, lambda b: received.append(b))
    # Z80 schreibt 0x0D (CR) an Drucker-SIO
    k8025_lib.k8025_io_write(card, 0x5C, 0x0D)  # SIO A32 Ch A Data
    assert 0x0D in received
```
