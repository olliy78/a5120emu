## 1. Aufgabe

Die K7024 ist die Bildschirmkarte des A5120. Sie generiert das Video-Signal für den angeschlossenen Phosphor-Monitor. Im Emulator schreibt sie in einen Software-Framebuffer, den die Python-GUI darstellt.

## 2. Hardware-Spezifikation

| Parameter | Wert |
|-----------|------|
| Zeichen/Zeile | 80 |
| Zeilen | 24 |
| Gesamtzeichen | 1920 |
| Pixel/Zeichen (B×H) | 8 × 12 |
| Framebuffer-Größe | 640 × 288 Pixel |
| Bildwiederholrate | 51 Hz |
| VRAM-Größe | 2048 Byte |
| VRAM-Adresse (A5120) | 0xF800–0xFFFF |
| Zeichengenerator | 2 × 1024 Byte EPROM |

---

## 3. VRAM-Layout

```
F800H–F84FH:  Zeile 0 (80 Bytes für 80 Zeichen)
F850H–F899H:  Zeile 1
...
FFB0H–FFF9H:  Zeile 23
FFFAH–FFFFH:  6 Bytes ungenutzt (Rest des 2KB-Blocks)
```

Jedes Byte im VRAM kodiert ein Zeichen:
- Bits 6:0 = ISO-7-Bit Zeichencode
- Bit 7 = Cursor-Flag (Zeile 11–12 wird als Cursor dargestellt)

**K7024.40 zusätzlich:**
- Bit 0 des vorherigen Bytes = Invers-Attribut (XOR mit VIDEO)
- Bit 1 = Intensiv-Attribut

---

## 4. Zeichengenerator-Adressierung

Der Zeichengenerator (1 KB EPROM) enthält 8 Bytes pro Zeichen (für Zeilen 0–7). Die obere/untere Hälfte der 12 Pixelzeilen wird durch das `/LP3`-Signal selektiert.

```
EPROM-Adresse = (zeichencode & 0x7F) << 3 | zeile
Adressbits:
  A9:A3 = Zeichencode (7 Bit, ISO)
  A2:A0 = Zeile innerhalb Zeichen (0–7)
  /LP3  = EPROM-Auswahl (0 = obere 8 Zeilen via A103, 1 = untere 4 via Zusatz-ROM)
```

**Vereinfachte Emulation (8×8 sichtbare Pixel aus 12 Zeilen):**

Da das EPROM nur 8 Zeilen pro Zeichen enthält (A2:A0 = 0–7), die tatsächliche Zeichenzelle aber 12 Pixelzeilen hat, werden die Zeilen wie folgt zugeordnet:
- Pixelzeile 0–7: direkt aus EPROM (Zeile 0–7)
- Pixelzeile 8–9: leer (Zeichenabstand)
- Pixelzeile 10–11: leer (Zeilenabstand) oder Cursor

```cpp
// Zeichengenerator-Lookup
static uint8_t chargenLookup(uint8_t charCode, int pixelRow,
                              const uint8_t* charset) {
    if (pixelRow >= 8) return 0x00;  // Abstands-Zeilen = leer
    uint16_t addr = ((charCode & 0x7F) << 3) | (pixelRow & 0x07);
    return (addr < 1024) ? charset[addr] : 0x00;
}
```

## 5. Klassen-Interface

```cpp
class K7024 : public MemDevice, public BusDevice {
public:
    // Konfiguration (Brücken X11/X12 für VRAM-Startadresse)
    struct A5120Config {
        uint16_t vram_base    = 0xF800;   // Standard A5120
        bool     cyrillic     = false;    // Kyrillischer Zeichensatz
        bool     invers_attr  = false;    // K7024.40: Invers-Attribut
        bool     cursor_blink = true;     // Cursor blinkt
    };

    K7024(K1520Bus& bus, Koppelbus& koppel, const A5120Config& cfg = {});

    // MemDevice: VRAM (2 KB ab vram_base)
    uint8_t memRead(uint16_t addr) override;
    void    memWrite(uint16_t addr, uint8_t data) override;

    // BusDevice: /RDY-Signal bei I/O-Zugriff (K1520-Bus-Standard)
    uint8_t ioRead(uint8_t port) override  { return 0xFF; }
    void    ioWrite(uint8_t port, uint8_t data) override {}

    // Karte an Bus anmelden
    void attachToBus(K1520Bus& bus);

    // ─── Framebuffer-Interface (für C-API) ─────────────────────

    // Pixel-Framebuffer: 640×288 Bytes, monochrom (0=schwarz, 255=weiß)
    const uint8_t* getFramebuffer() const;
    int            getFramebufferWidth() const  { return 640; }
    int            getFramebufferHeight() const { return 288; }
    bool           isFramebufferDirty() const;
    void           clearDirty();

    // ─── Text-Interface (CLI-Modus) ─────────────────────────────

    // Prüft ob ein Zeichen im VRAM geändert wurde
    // Gibt true zurück und setzt x,y,ch wenn eine Änderung vorliegt
    bool    pollTextChange(int& x, int& y, char& ch);
    void    getCursorPos(int& x, int& y) const;

    // ─── Zeichensatz-Umschaltung ─────────────────────────────────
    void    setCharset(bool cyrillic);

private:
    A5120Config cfg_;
    uint8_t     vram_[2048]{};
    uint8_t     framebuffer_[640 * 288]{};
    bool        dirty_     = false;
    bool        cyrillic_  = false;
    int         cursor_x_  = 0;
    int         cursor_y_  = 0;

    // Text-Change-Queue (CLI-Modus)
    struct TextChange { int x, y; char ch; };
    std::queue<TextChange> text_changes_;

    // Zeichensatz-Pointer (Latin oder Cyrillic)
    const uint8_t* charset_ = CHARSET_LATIN;

    void renderChar(int col, int row);
    void renderAll();
    void renderCursor();
    void updateCursorPos();  // Cursor aus VRAM-Bit 7 ermitteln
};
```

## 6. Framebuffer-Rendering

```cpp
void K7024::renderChar(int col, int row) {
    uint8_t charByte = vram_[row * 80 + col];
    bool    isCursor = (charByte & 0x80) != 0;
    uint8_t charCode = charByte & 0x7F;

    int pixelX = col * 8;
    int pixelY = row * 12;

    for (int py = 0; py < 12; py++) {
        uint8_t pixels = chargenLookup(charCode, py, charset_);

        for (int px = 0; px < 8; px++) {
            bool lit = (pixels >> (7 - px)) & 1;

            // Cursor: Pixelzeilen 10–11 invertieren
            if (isCursor && py >= 10) lit = !lit;

            framebuffer_[(pixelY + py) * 640 + pixelX + px] =
                lit ? 0xFF : 0x00;
        }
    }
}
```

## 7. CLI-Textmodus

Im CLI-Modus werden VRAM-Änderungen als Unicode-Zeichen auf stdout geschrieben, statt in den Framebuffer gerendert.

```cpp
void K7024::memWrite(uint16_t addr, uint8_t data) {
    if (addr < cfg_.vram_base || addr >= cfg_.vram_base + 2048) return;
    uint16_t offset = addr - cfg_.vram_base;
    if (vram_[offset] == data) return;
    vram_[offset] = data;

    int col = offset % 80;
    int row = offset / 80;

    if (console_mode_) {
        // CLI: Zeichen in Queue einreihen
        char ch = (char)(data & 0x7F);
        if (ch < 0x20) ch = ' ';  // Steuerzeichen → Leerzeichen
        text_changes_.push({col, row, ch});
    } else {
        // GUI: Framebuffer aktualisieren
        renderChar(col, row);
        dirty_ = true;
    }
}
```

## 8. Cursor-Emulation

Der K7024 rendert den Cursor in Pixelzeilen 10–11 des zugehörigen Zeichens. Das Cursor-Zeichen ist das Zeichen mit gesetztem Bit 7 im VRAM.

Blinken (51 Hz / 2 ≈ 0.5 Hz): Die Python-GUI implementiert das Blinken durch alternierendes Rendern des Cursors im `ScreenWidget`.

## 9. Schnittstelle zur Python-GUI

```python
# In ScreenWidget.paintEvent:
fb = k1520.framebuffer()          # 640×288 bytes (monochrom)
img = QImage(fb, 640, 288, QImage.Format_Grayscale8)

# Phosphor-Farbe applizieren (A5120: grün)
rgb = img.convertToFormat(QImage.Format_RGB32)
for y in range(288):
    for x in range(640):
        gray = qGray(rgb.pixel(x, y))
        green = qRgb(gray // 4, gray, gray // 8)  # P31-Phosphor
        rgb.setPixel(x, y, green)

# Ganzzahlige Skalierung (2×)
scaled = rgb.scaled(1280, 576, Qt.KeepAspectRatio, Qt.FastTransformation)
QPainter(self).drawImage(0, 0, scaled)
```

```python
# tests/python/test_k7024.py
def test_char_render_A(k7024_lib):
    screen = k7024_lib.k7024_create()
    # Zeichen 'A' (0x41) in Spalte 0, Zeile 0 schreiben
    k7024_lib.k7024_vram_write(screen, 0xF800, 0x41)
    fb = k7024_lib.k7024_get_framebuffer(screen)
    # Pixel (0,0) muss gesetzt sein (erstes Pixel von 'A')
    # Prüfung hängt vom tatsächlichen EPROM-Inhalt ab
    assert fb[0] != 0x00  # Zumindest ein Pixel gesetzt

def test_cursor_flag(k7024_lib):
    screen = k7024_lib.k7024_create()
    # Zeichen mit Cursor-Bit (Bit 7 = 1)
    k7024_lib.k7024_vram_write(screen, 0xF800, 0x41 | 0x80)
    cx, cy = k7024_lib.k7024_get_cursor_pos(screen)
    assert cx == 0 and cy == 0
```
