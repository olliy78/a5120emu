## 1. Aufgabe

Die K2526 ist die Hauptprozessorkarte des A5120. Sie enthält:
- **ZVE1**: Haupt-Z80-CPU (U880D, 2.4576 MHz)
- **ZVE2**: Zweite Z80-CPU als DMA-Prozessor
- **Lade-ROM**: 1 KB EPROM bei 0000H–03FFH
- **CTC** (Q 302): 4 Kanäle, Timer/Zähler, I/O 0CH–0FH
- **BS-PIO** (Q 301): Steuerfunktionen, I/O 08H–0BH
- **U-Bus**: 4 spezielle I/O-Adressen (00H–07H)
- **Speicher-/E/A-Schutz** (Q 240)
- **Taktgenerator**: 9832 kHz Quarz → 2.4576 MHz

---

## 2. Interne Architektur

```
K2526-Karte
├── Z80CPU (ZVE1)           – Haupt-CPU, läuft auf K1520Bus
├── Z80CPU (ZVE2)           – DMA-CPU (läuft auf K1520Bus wenn ZVE1 schläft)
├── Z80CTC                  – Ports 0CH-0FH
├── Z80PIO (BS-PIO)         – Ports 08H-0BH
├── EPROMDevice (1KB)       – Lade-ROM, 0000H-03FFH
├── UBus-Register           – Ports 00H-07H (U 216 + U 212)
└── MemIOProtect            – /MEMDI-Logik (Q 240)
```

class K2526 : public BusDevice {
public:

    struct A5120Config {

    // Koppelbus-Schnittstellen

    // U-Bus (für Tests / BS-PIO-Emulation)

    void    writeUCS4(uint8_t data);   // Port 03H: Fehlerlampe

    // Lade-ROM Steuerung (BS-PIO B0)
    bool    isRomEnabled() const;

    // Callbacks (für übergeordnete Maschine)

    void    setNMICallback(NMICallback);

private:
    K1520Bus&    bus_;
    Koppelbus&   koppel_;
    A5120Config  cfg_;

    Z80CPU       zve1_{bus_};           // Haupt-CPU
    Z80CPU       zve2_{bus_};           // DMA-CPU
    Z80CTC       ctc_{"K2526-CTC"};
    Z80PIO       bs_pio_{"K2526-BSPIO"};
    EPROMDevice<1024> rom_{ZRE_BOOT_ROM};  // aus rom_data.h

    // U-Bus Register
    uint8_t      ubus_kbd_code_   = 0xFF;  // UCS1: Tastencode
    uint8_t      ubus_kbd_valid_  = 0x00;  // UCS2: Gültig?
    uint8_t      ubus_lamp_       = 0x00;  // UCS3: Lampen
    uint8_t      ubus_error_lamp_ = 0x00;  // UCS4: Fehlerlampe

    bool         rom_enabled_ = true;   // /LD-ROM (BS-PIO B0)
    bool         iei1_        = true;

## 4. I/O-Port-Mapping (00H–0FH)

| Port | Richtung | Funktion | Realisierung |
|------|----------|----------|--------------|
| 00H | R/W | /INT-BS (BS-PIO-Interrupt) | → BS-PIO A0-Interrupt |
| 01H | R | /UCS2: Tastengültigkeitsabfrage | UBus-Register |
| 02H | W | /RES-SPA (Speicherschutz Reset) | MemIOProtect |
| 03H | W | /UCS4: Fehlerlampe | UBus-Register |
| 04H | W | /RES-ZVE2: DMA-CPU Reset | ZVE2.reset() |
| 05H | W | /UCS3: Lampenansteuerung | UBus-Register |
| 06H | R | /UCS1: Tastencode lesen | UBus-Register |
| 07H | — | reserviert | — |
| 08H | R/W | BS-PIO Port A Daten | → Z80PIO |
| 09H | R/W | BS-PIO Port A Steuer | → Z80PIO |
| 0AH | R/W | BS-PIO Port B Daten | → Z80PIO |
| 0BH | R/W | BS-PIO Port B Steuer | → Z80PIO |
| 0CH | R/W | CTC Kanal 0 | → Z80CTC |
| 0DH | R/W | CTC Kanal 1 | → Z80CTC |
| 0EH | R/W | CTC Kanal 2 | → Z80CTC |
| 0FH | R/W | CTC Kanal 3 | → Z80CTC |

## 5. BS-PIO Funktionen

### 5.1 Port A (Eingänge)

| Bit | Signal | Beschreibung |
|-----|--------|--------------|
| 0 | /M1 | Einzelbefehlsabarbeitung (Interrupt-Vektor für ZVE1-Einzelschritt) |
| 1 | /SUE | Spannungsüberwachungseingang |
| 2 | /NMI | NMI-Quelle (Netzausfall / E/A-Schutz-Verletzung) |
| 3 | SPS | Speicherschutz-Verletzungsanzeige |
| 4 | /EBF | Einzelbefehlsende |
| 5 | /WR | Speichertest (WR-Signal für Speicherkarten-Test) |
| 6 | /RDY | Speicher/Peripherie bereit |
| 7 | /MEMDI1/2 | Speicherbereich-Umschaltsignal (Ausgang!) |

### 5.2 Port B (Ausgänge)

| Bit | Signal | Beschreibung | Emulation |
|-----|--------|--------------|-----------|
| 0 | /LD-ROM | Lade-ROM abschalten | `rom_enabled_ = !(data & 1)` |
| 1 | /INT-BS | OS-Ebenen-Wechsel anfordern | NMI/Interrupt an ZVE1 |
| 2 | /SPS-ESR | Speicherschutz-RAM Schreibgenehmigung | MemIOProtect |
| 3 | /WAIT-ZVE2 | WAIT-Signal für ZVE2 | ZVE2-WAIT |
| 4 | /SA | Sonderausgang (Netzausschaltung) | Koppelbus.sa |
| 5–7 | Config | Brückentest (Eingabe der Konfiguration) | cfg_.config_bridges |

### 5.3 Lade-ROM Abschaltung

Beim Systemstart ist das 1 KB Lade-ROM bei 0000H–03FFH aktiv. Das BIOS schreibt nach der Initialisierung Bit 0 = 1 in BS-PIO Port B, um das ROM abzuschalten. Danach liegt das RAM des K3526 vollständig bei 0000H.

```cpp
void K2526::onBSPIOPortBOut(uint8_t data) {
    bool rom_disable = (data & 0x01) != 0;
    if (rom_disable && rom_enabled_) {
        bus_.unregisterMem(&rom_);
        rom_enabled_ = false;
    }
    // /SA: Netzausschaltung → Koppelbus-Signal
    koppel_.sa.drive((data & 0x10) == 0);
}
```

## 6. ZVE2 (DMA-CPU)

Die zweite CPU (ZVE2) funktioniert als DMA-Prozessor. Im A5120 wird sie hauptsächlich für:
- **Diskettenoperationen**: Datenübertragung zwischen FDC und RAM
- **Speichertests**: RAM-Test im Bootvorgang

**Emulationsansatz:**
- ZVE2 ist ebenfalls ein vollständiger Z80CPU
- ZVE2 läuft nur, wenn ZVE1 `/BUSRQ` gesetzt hat (Bus freigegeben)
- ZVE1 und ZVE2 teilen sich den K1520Bus; der Bus-Simulator schaltet zwischen ihnen um
- `/RES-ZVE2` (Port 04H) setzt ZVE2 zurück

```cpp
int K2526::runCycles(int cycles) {
    if (!bus_.isBUSRQ()) {
        return zve1_.run(cycles);
    } else {
        // ZVE1 wartet, ZVE2 hat Bus
        return zve2_.run(cycles);
    }
}
```

## 7. Speicher-/E/A-Schutz (Q 240)

Die Q240-Schutzlogik teilt den 64KB-Adressraum in 1024 Segmente à 64 Bytes. Ein Segment-Schreibschutz-RAM gibt an, welche Segmente geschützt sind. Bei Verletzung:
- Schreib-/Lesezugriff auf geschütztes Segment → `/MEMDI` setzt Bus
- Verbotener I/O-Befehl → NMI wird ausgelöst

**Emulationsansatz:** Vereinfacht implementiert als Schreibschutz-Bitmap (1024 Bits). Geschützte Bereiche ignorieren Schreibzugriffe und können optional NMI auslösen.

```cpp
class MemIOProtect {
public:
    void setProtect(uint16_t addr, bool protect);
    bool isProtected(uint16_t addr) const;
    void reset();

private:
    std::bitset<1024> protected_segments_;
};
```

## 8. Boot-Sequenz

```
/RESET → ZVE1 startet bei 0000H
         ROM aktiv (1KB bei 0000H)
         ROM-Code:
           1. RAM-Test (K3526)
           2. CTC initialisieren (Baud-Rate)
           3. Ersten Sektor von Diskette lesen (K5122)
           4. Bootstrap-Code bei 0100H ausführen
           5. BS-PIO B0 setzen → ROM abschalten
           6. Betriebssystem läuft nativ
```

Das Lade-ROM (1 KB, `doc/EPROMS/Prozessoreinheit_ZRE_K2525_A26.bin`) enthält den oben beschriebenen Bootstrap-Code.

## 9. Testbarkeit

```python
# tests/python/test_k2526.py
def test_rom_enabled_after_reset(k2526_lib):
    machine = k2526_lib.k2526_create_standalone()
    k2526_lib.k2526_reset(machine)
    # ROM bei 0x0000 muss lesbar sein
    assert k2526_lib.k2526_mem_read(machine, 0x0000) != 0xFF

def test_rom_disabled_by_bspio(k2526_lib):
    machine = k2526_lib.k2526_create_standalone()
    k2526_lib.k2526_reset(machine)
    # BS-PIO Port B = 0x01 → /LD-ROM abschalten
    k2526_lib.k2526_io_write(machine, 0x0A, 0x01)
    # ROM-Bereich jetzt RAM
    k2526_lib.k2526_mem_write(machine, 0x0000, 0xAB)
    assert k2526_lib.k2526_mem_read(machine, 0x0000) == 0xAB
```
