**Modul:** `core/cards/k2526/`  
**Dateien:** `k2526.h`, `k2526.cpp`, `rom_data.h` (generiert)  
**Dokumentation:** `doc/trascripted/Zentrale Recheneinheit_K2526_K2527.md`

---

## 1. Aufgabe

Die K2526 ist die Hauptprozessorkarte des A5120. Sie enthält:
- **ZVE1**: Haupt-Z80-CPU (U880D, 2.4576 MHz)
- **ZVE2**: Zweite Z80-CPU als DMA-Prozessor
- **Lade-ROM**: 1 KB EPROM bei 0000H–03FFH
- **CTC** (Q 302): 4 Kanäle, Timer/Zähler, I/O 0CH–0FH
- **BS-PIO** (Q 301): Steuerfunktionen, I/O 08H–0BH
- **U-Bus**: 4 spezielle I/O-Adressen (00H–07H)
- **Speicher-/E/A-Schutz** (Q 240)
- **Taktgenerator**: 9832 kHz Quarz → 2.4576 MHz

**Emulationsumfang:** Die K2526-Klasse emuliert Lade-ROM, BS-PIO, CTC und U-Bus-Register.
ZVE1 (Haupt-CPU) wird als separates Z80-Objekt in `A5120Machine` gehalten und nicht innerhalb von K2526.
ZVE2 und der Speicher-/E/A-Schutz (Q240) sind **nicht implementiert**.

---

## 2. Interne Architektur

```
K2526-Karte
├── Z80CPU (ZVE1)           – Haupt-CPU (liegt in A5120Machine, nicht in K2526)
├── Z80CPU (ZVE2)           – DMA-CPU (nicht implementiert)
├── Z80CTC                  – Ports 0CH-0FH
├── Z80PIO (BS-PIO)         – Ports 08H-0BH
├── EPROMDevice (1KB)       – Lade-ROM, 0000H-03FFH (abschaltbar)
├── UBus-Register           – Ports 00H-07H (Tastatur, Lampen)
└── MemIOProtect (Q240)     – nicht implementiert
```

## 3. Klassen-Interface

```cpp
class K2526 : public BusDevice, public InterruptSlave {
public:
    struct A5120Config {
        uint8_t io_base        = 0x00;   // I/O-Basis (fest 00H im A5120)
        uint8_t config_bridges = 0x00;   // BS-PIO B5-B7 Brücken-Codierung
    };

    explicit K2526(K1520Bus& bus);
    K2526(K1520Bus& bus, const A5120Config& cfg);

    // BusDevice: I/O-Ports 00H-0FH
    uint8_t     ioRead(uint8_t port) override;
    void        ioWrite(uint8_t port, uint8_t data) override;
    const char* deviceName() const override { return "K2526"; }

    // InterruptSlave (interne Kette: IEI → CTC → BS-PIO → IEO)
    void    setIEI(bool iei) override;
    bool    getIEO() const override;
    bool    hasInterrupt() const override;
    uint8_t getVector() const override;
    void    onRETI() override;

    // Lifecycle
    void attachToBus(K1520Bus& bus);   // Ports 00H-0FH registrieren
    void powerOn();                    // Lade-ROM einblenden
    void clockTick();                  // CTC-Takt (einmal pro CPU-Step)

    // Status-Abfrage
    bool isRomEnabled() const { return rom_enabled_; }

    // Sub-Chip-Accessoren
    Z80PIO& bsPio() { return bs_pio_; }
    Z80CTC& ctc()   { return ctc_; }

    // U-Bus-Schnittstelle (Tastatur-Eingabe von außen)
    void setKbdCode(uint8_t code)  { ubus_kbd_code_  = code; }
    void setKbdValid(uint8_t val)  { ubus_kbd_valid_ = val;  }

private:
    void onBSPIOPortBOut(uint8_t data);

    K1520Bus&   bus_;
    A5120Config cfg_;

    Z80PIO bs_pio_{"K2526-BS-PIO"};
    Z80CTC ctc_   {"K2526-CTC"};

    EPROMDevice<1024> rom_{ZRE_BOOT_ROM};

    bool    rom_enabled_ = true;
    bool    iei_in_      = false;

    // U-Bus-Register
    uint8_t ubus_kbd_code_   = 0xFF;  // Port 06H IN:  Tastencode (/UCS1)
    uint8_t ubus_kbd_valid_  = 0x00;  // Port 01H IN:  Gültig?    (/UCS2)
    uint8_t ubus_lamp_       = 0x00;  // Port 05H OUT: Lampen     (/UCS3)
    uint8_t ubus_error_lamp_ = 0x00;  // Port 03H OUT: Fehlerlampe(/UCS4)
};
```

## 4. I/O-Port-Mapping (00H–0FH)

| Port | Richtung | Funktion | Realisierung |
|------|----------|----------|--------------|
| 00H | — | reserviert (urspr. /INT-BS) | ignoriert |
| 01H | R | /UCS2: Tastengültigkeitsabfrage | `ubus_kbd_valid_` |
| 02H | W | /RES-SPA (Speicherschutz Reset) | nicht implementiert |
| 03H | W | /UCS4: Fehlerlampe | `ubus_error_lamp_` |
| 04H | W | /RES-ZVE2: DMA-CPU Reset | nicht implementiert |
| 05H | W | /UCS3: Lampenansteuerung | `ubus_lamp_` |
| 06H | R | /UCS1: Tastencode lesen | `ubus_kbd_code_` |
| 07H | — | reserviert | ignoriert |
| 08H | R/W | BS-PIO Port A Daten | → Z80PIO |
| 09H | R/W | BS-PIO Port A Steuer | → Z80PIO |
| 0AH | R/W | BS-PIO Port B Daten | → Z80PIO |
| 0BH | R/W | BS-PIO Port B Steuer | → Z80PIO |
| 0CH | R/W | CTC Kanal 0 | → Z80CTC |
| 0DH | R/W | CTC Kanal 1 | → Z80CTC |
| 0EH | R/W | CTC Kanal 2 | → Z80CTC |
| 0FH | R/W | CTC Kanal 3 | → Z80CTC |

## 5. BS-PIO Funktionen

### 5.1 Port A (Richtungsmaske 0x7F – Bit 7 ist Ausgang)

| Bit | Signal | Beschreibung |
|-----|--------|--------------|
| 0 | /M1 | Einzelbefehlsabarbeitung (Interrupt-Vektor für ZVE1-Einzelschritt) |
| 1 | /SUE | Spannungsüberwachungseingang |
| 2 | /NMI | NMI-Quelle (Netzausfall / E/A-Schutz-Verletzung) |
| 3 | SPS | Speicherschutz-Verletzungsanzeige |
| 4 | /EBF | Einzelbefehlsende |
| 5 | /WR | Speichertest (WR-Signal für Speicherkarten-Test) |
| 6 | /RDY | Speicher/Peripherie bereit |
| 7 | MEMDI1/2 | Speicherbereich-Umschaltsignal **(Ausgang!)**: A7=1 → `bus_.setMEMDI(true)` |

### 5.2 Port B (Ausgänge)

| Bit | Signal | Beschreibung | Emulation |
|-----|--------|--------------|-----------|
| 0 | /LD-ROM | Lade-ROM abschalten | `rom_enabled_ = (data & 1)` |
| 1 | /INT-BS | OS-Ebenen-Wechsel anfordern | nicht implementiert |
| 2 | /SPS-ESR | Speicherschutz-RAM Schreibgenehmigung | nicht implementiert |
| 3 | /WAIT-ZVE2 | WAIT-Signal für ZVE2 | nicht implementiert |
| 4 | /SA | Sonderausgang (Netzausschaltung) | nicht implementiert |
| 5–7 | Config | Brückentest (Eingabe der Konfiguration) | `cfg_.config_bridges` |

### 5.3 Lade-ROM Abschaltung

Beim Systemstart ist das 1 KB Lade-ROM bei 0000H–03FFH aktiv (B0 per Pull-Up auf 1).
Das BIOS schreibt nach der Initialisierung **Bit 0 = 0** (B0=0, /LD-ROM aktiviert) in BS-PIO
Port B, um das ROM abzuschalten. Danach liegt das RAM des K3526 vollständig bei 0000H.

```cpp
void K2526::onBSPIOPortBOut(uint8_t data) {
    // /LD-ROM ist active-low: B0=1 → ROM aktiv, B0=0 → ROM abgeschaltet
    bool rom_on = (data & 0x01) != 0;
    if (!rom_on && rom_enabled_) {
        bus_.unregisterMem(&rom_);
        rom_enabled_ = false;
        LOG_INFO("K2526", "Lade-ROM abgeschaltet (B0=0)");
    }
    // Idempotenz: zweiter Write mit B0=0 hat keinen Effekt
}
```

## 6. ZVE2 (DMA-CPU) – nicht implementiert

Die zweite CPU (ZVE2) funktioniert als DMA-Prozessor und wurde hauptsächlich für
Diskettenoperationen und Speichertests verwendet. Im A5120-BIOS läuft ZVE2 als
Koprozessor, während ZVE1 mit `/BUSRQ` den Bus freigibt.

**Status im Emulator:** ZVE2 ist nicht implementiert. Die K5122-Karte verwendet
stattdessen ein vereinfachtes Byte-Polling-Modell ohne DMA/BUSRQ-Mechanismus.
Damit entfallen auch `/RES-ZVE2` (Port 04H) und `/WAIT-ZVE2` (BS-PIO B3).

---

## 7. Speicher-/E/A-Schutz (Q 240) – nicht implementiert

Die Q240-Schutzlogik unterteilt den 64KB-Adressraum in 1024 Segmente à 64 Bytes
und schützt einzelne Segmente vor unberechtigtem Schreib-/Lesezugriff.

**Status im Emulator:** Nicht implementiert. Alle Speicherzugriffe werden ungeprüft
durchgeführt. Die Ports `/RES-SPA` (02H) und `/SPS-ESR` (BS-PIO B2) sind No-Ops.

---

## 8. Boot-Sequenz

```
/RESET → ZVE1 startet bei 0000H
         ROM aktiv (1KB bei 0000H, read-only; Schreibzugriffe gehen an K3526-RAM)
         ROM-Code:
           1. RAM-Test (K3526)
           2. CTC initialisieren (Baud-Rate für SIO auf K8025)
           3. Ersten Sektor von Diskette lesen (K5122)
           4. Bootstrap-Code bei 0100H ausführen
           5. BS-PIO B0 auf 0 setzen → /LD-ROM aktiv → ROM abschalten
           6. Betriebssystem läuft nativ im RAM
```

Das Lade-ROM (1 KB, `doc/EPROMS/Prozessoreinheit_ZRE_K2525_A26.bin`) enthält den
oben beschriebenen Bootstrap-Code.

## 9. Testbarkeit

```cpp
// tests/cpp/test_k2526.cpp

TEST(K2526, PowerOn_RomData_MatchesArray) {
    K1520Bus bus;
    K2526 card(bus);
    card.attachToBus(bus);
    card.powerOn();
    for (uint16_t i = 0; i < 1024; ++i)
        EXPECT_EQ(bus.memRead(i), ZRE_BOOT_ROM[i]);
}

TEST(K2526, BSPioPortB_Bit0Low_DisablesRom) {
    K1520Bus bus;
    K2526 card(bus);
    card.attachToBus(bus);
    card.powerOn();
    // B0=0 → /LD-ROM assertiert (active low) → ROM abschalten
    bus.ioWrite(0x0A, 0x00);
    EXPECT_FALSE(card.isRomEnabled());
    EXPECT_EQ(bus.memRead(0x0000), 0xFF);  // kein Device → open drain
}

TEST(K2526, BSPioPortA_Bit7_SetsMEMDI) {
    K1520Bus bus;
    K2526 card(bus);
    card.attachToBus(bus);
    card.powerOn();
    bus.ioWrite(0x08, 0x80);   // Port A: A7=1 → MEMDI aktiv
    EXPECT_TRUE(bus.getMEMDI());
    bus.ioWrite(0x08, 0x00);   // A7=0 → MEMDI inaktiv
    EXPECT_FALSE(bus.getMEMDI());
}
```
