# Feinentwurf: K5122 AFS – Floppy-Anschlußsteuerung

**Modul:** `core/cards/k5122/`  
**Dateien:** `k5122.h`, `k5122.cpp`  
**Dokumentation:** `doc/trascripted/Floppy Anschlußsteuerung K 5122.md`

---

## 1. Aufgabe

Die K5122 steuert bis zu 4 Diskettenlaufwerke. Sie ist keine direkte FDC-Implementierung wie der µPD765, sondern eine **PIO-basierte Lösung**: Zwei Z80-PIO-Chips übernehmen die Bit-by-Bit-Kommunikation mit den Laufwerken unter Steuerung der CPU.

Im Emulator wird die analoge Diskettenkommunikation durch **direkten Image-Zugriff** ersetzt.

---

## 2. Hardware-Zusammensetzung

```
K5122-Karte
├── Steuer-PIO A1.2 (Z80-PIO) – Ports 10H–13H
│   ├── Tor A Output: Steuersignale (/WE, MK, /STR, /ST, ...)
│   └── Tor B BitMode: Status-Ein-/Ausgänge (/RDYL, /TO, /WP, ...)
├── Daten-PIO A1.1 (Z80-PIO) – Ports 14H–17H
│   ├── Tor A Output: Schreibdaten (8 Bit)
│   └── Tor B Input: Lesedaten (8 Bit)
└── Laufwerksauswahl A4 (8212) – Port 18H
    └── /LCK1-4, /SE3, /SE4, /SET, /SEZ
```

## 3. I/O-Port-Belegung

| Port | PIO | Tor | Typ | Funktion |
|------|-----|-----|-----|----------|
| 10H | S | A | Data | Steuerausgänge |
| 11H | S | A | Ctrl | PIO-Steuerregister |
| 12H | S | B | Data | Status-Eingang/Ausgang |
| 13H | S | B | Ctrl | PIO-Steuerregister |
| 14H | D | A | Data | Schreibdaten (8 Bit) |
| 15H | D | A | Ctrl | PIO-Steuerregister |
| 16H | D | B | Data | Lesedaten (8 Bit) |
| 17H | D | B | Ctrl | PIO-Steuerregister |
| 18H | — | — | — | Laufwerksauswahl (8212) |

---

## 4. Steuersignale (Steuer-PIO Tor A Ausgänge)

| Bit | Signal | Funktion |
|-----|--------|----------|
| 0 | /WE | Write Enable (Schreibkopf aktivieren) |
| 1 | MK | Mark (Startmarkierung) |
| 2 | /FR | Fault Reset (Fehler rücksetzen) |
| 3 | /STR | Start (Kommando starten) |
| 4 | MK1 | Mark 1 |
| 5 | MR/SD | Mark Reset / Step Direction (0=inward, 1=outward) |
| 6 | /HL | Head Load (Schreib-/Lesekopf aufsetzen) |
| 7 | /ST | Step (ein Schritt in Richtung MR/SD) |

## 5. Status-Signale (Steuer-PIO Tor B Eingänge/Ausgänge)

| Bit | Signal | Richtung | Funktion |
|-----|--------|----------|----------|
| 0 | /RDYL | Eingang | Laufwerk bereit (MFS) |
| 1 | MKE | Eingang | Marke erkannt |
| 2 | /HF | Ausgang | High Frequency (0=MFM 8", 1=FM 5") |
| 3 | PRE | Ausgang | Precompensation |
| 4 | /FA | Eingang | Fault Adapter |
| 5 | /WP | Eingang | Write Protect |
| 6 | /FW | Eingang | Fault (Laufwerksfehler) |
| 7 | /TO | Eingang | Track 00 (auf Spur 0) |

## 7. Vereinfachtes Transfer-Modell (kein DMA/BUSRQ)

Das originale A5120-System überträgt Sektordaten über ZVE2 als DMA-CPU mit BUSRQ/BUSACK.
Im Emulator wird dies durch ein **byte-orientiertes Polling-Modell** ersetzt, da ZVE2 nicht
implementiert ist.

### 7.1 SEEK-Operation

Ein Schritt-Puls (/ST: Bit 7 = fallende Flanke) bewegt den Kopf in Richtung MR/SD (Bit 5).

```
CPU schreibt 0x7F an Port 0x10 (prev_ctrl_a_ = 0xFF):
  /ST (Bit 7) = 0:  fallende Flanke → doStep()
  MR/SD (Bit 5) = 1: outward (sinkende Spur)
→ current_cyl_[drive]--
→ updateStatusPortB(): /FW = 0 wenn Spur 0 erreicht
```

### 7.2 LESE-Operation (Byte-Polling)

```
1. CPU schreibt /STR=0 (Bit 3) mit /WE=1 (Bit 0) → doReadSector()
   → Sektor aus Disk-Image in sector_buf_ laden
   → transferring_ = true, write_mode_ = false
2. CPU liest Port 0x16 (Daten-PIO Port B) wiederholt:
   → each ioRead(0x16): nächstes Byte aus sector_buf_[buf_pos_++]
   → nach letztem Byte: transferring_ = false
   → current_sector_ automatisch erhöht (wrapping an secs_per_track)
```

### 7.3 SCHREIB-Operation (Byte-Polling)

```
1. CPU schreibt Bytes an Port 0x14 (Daten-PIO Port A):
   → each ioWrite(0x14, byte): sector_buf_.push_back(byte)
2. CPU schreibt /STR=0 (Bit 3) mit /WE=0 (Bit 0) → doWriteSector()
   → sector_buf_ in Disk-Image schreiben
   → current_sector_ automatisch erhöht
   → sector_buf_.clear()
```

### 7.4 Laufwerksstatus (ctrl_pio_ Port B)

Nach jeder Statusänderung wird `updateStatusPortB()` aufgerufen, das den Statusbyte
in den Eingangslatch von ctrl_pio_ schreibt. Die CPU liest den Status über Port 0x12.

```
Kein Laufwerk: 0xF5 = /TO=1, /FW=1, /WP=1, /FA=1, MKE=0, /RDYL=1
Laufwerk eingelegt: /RDYL=0 (bereit)
Auf Spur 0: /FW=0
Schreibgeschützt: /WP=0
```

## 8. Laufwerksauswahl (8212, Port 18H)

```
Bit-Belegung Port 18H (Schreibzugriff):
  DO1 = /LCK2   DO5 = /LCK3
  DO2 = /LCK1   DO6 = /LCK4
  DO3 = /SE3    DO7 = /SET
  DO4 = /SE4    DO8 = /SEZ

Laufwerksauswahl durch:
  /LCK1 = 0 → Laufwerk A (drive 0)
  /LCK2 = 0 → Laufwerk B (drive 1)
  /LCK3 = 0 → Laufwerk C (drive 2)
  /LCK4 = 0 → Laufwerk D (drive 3)
```

def test_seek_and_status(k5122_lib, tmp_cpa780_image):
    card = k5122_lib.k5122_create()

    k5122_lib.k5122_io_write(card, 0x10, 0b10111111)

    k5122_lib.k5122_io_write(card, 0x10, 0b01111111)
