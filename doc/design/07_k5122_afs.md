# Feinentwurf: K5122 AFS – Floppy-Anschlußsteuerung

**Modul:** `core/cards/k5122/`  
**Dateien:** `k5122.h`, `k5122.cpp`  
**Dokumentation:** `doc/trascripted/Floppy Anschlußsteuerung K 5122.md`

---

## 1. Aufgabe

Die K5122 steuert bis zu 4 Diskettenlaufwerke. Sie ist keine direkte FDC-Implementierung wie der µPD765, sondern eine **PIO-basierte Lösung**: Zwei Z80-PIO-Chips übernehmen die Bit-by-Bit-Kommunikation mit den Laufwerken unter Steuerung der CPU.

Im Emulator wird die analoge Diskettenkommunikation durch **direkten Image-Zugriff** ersetzt.

---

## 2. Hardware-Zusammensetzung

```
K5122-Karte
├── Steuer-PIO A1.2 (Z80-PIO) – Ports 10H–13H
│   ├── Tor A Output: Steuersignale (/WE, MK, /STR, /ST, ...)
│   └── Tor B BitMode: Status-Ein-/Ausgänge (/RDYL, /TO, /WP, ...)
├── Daten-PIO A1.1 (Z80-PIO) – Ports 14H–17H
│   ├── Tor A Output: Schreibdaten (8 Bit)
│   └── Tor B Input: Lesedaten (8 Bit)
└── Laufwerksauswahl A4 (8212) – Port 18H
    └── /LCK1-4, /SE3, /SE4, /SET, /SEZ
```

## 3. I/O-Port-Belegung

| Port | PIO | Tor | Typ | Funktion |
|------|-----|-----|-----|----------|
| 10H | S | A | Data | Steuerausgänge |
| 11H | S | A | Ctrl | PIO-Steuerregister |
| 12H | S | B | Data | Status-Eingang/Ausgang |
| 13H | S | B | Ctrl | PIO-Steuerregister |
| 14H | D | A | Data | Schreibdaten (8 Bit) |
| 15H | D | A | Ctrl | PIO-Steuerregister |
| 16H | D | B | Data | Lesedaten (8 Bit) |
| 17H | D | B | Ctrl | PIO-Steuerregister |
| 18H | — | — | — | Laufwerksauswahl (8212) |

## 4. Steuersignale (Steuer-PIO Tor A Ausgänge)

| Bit | Signal | Funktion |
|-----|--------|----------|
| 0 | /WE | Write Enable (Schreibkopf aktivieren) |
| 1 | MK | Mark (Startmarkierung) |
| 2 | /FR | Fault Reset (Fehler rücksetzen) |
| 3 | /STR | Start (Kommando starten) |
| 4 | MK1 | Mark 1 |
| 5 | MR/SD | Mark Reset / Step Direction (0=inward, 1=outward) |
| 6 | /HL | Head Load (Schreib-/Lesekopf aufsetzen) |
| 7 | /ST | Step (ein Schritt in Richtung MR/SD) |

## 5. Status-Signale (Steuer-PIO Tor B Eingänge/Ausgänge)

| Bit | Signal | Richtung | Funktion |
|-----|--------|----------|----------|
| 0 | /RDYL | Eingang | Laufwerk bereit (MFS) |
| 1 | MKE | Eingang | Marke erkannt |
| 2 | /HF | Ausgang | High Frequency (0=MFM 8", 1=FM 5") |
| 3 | PRE | Ausgang | Precompensation |
| 4 | /FA | Eingang | Fault Adapter |
| 5 | /WP | Eingang | Write Protect |
| 6 | /FW | Eingang | Fault (Laufwerksfehler) |
| 7 | /TO | Eingang | Track 00 (auf Spur 0) |

## 6. Klassen-Interface

```cpp
class K5122 : public BusDevice, public InterruptSlave {
public:
    struct A5120Config {
        uint8_t io_base    = 0x10;     // Standard: 10H
        bool    use_busrq  = true;     // BUSRQ-Modus (Standard für A5120)
        int     num_drives = 3;        // A5120: 3 Laufwerke (A, B, C)
    };

    K5122(K1520Bus& bus, Koppelbus& koppel, const A5120Config& cfg = {});

    // BusDevice: Ports 10H–18H (9 Ports)
    uint8_t ioRead(uint8_t port) override;
    void    ioWrite(uint8_t port, uint8_t data) override;

    // InterruptSlave
    void    setIEI(bool) override;
    bool    getIEO() const override;
    bool    hasInterrupt() const override;
    uint8_t getVector() const override;

    // Laufwerksverwaltung (von C-API aufgerufen)
    bool    mountDrive(int drive, FloppyDrive* floppy);
    bool    unmountDrive(int drive);
    bool    isDriveActive(int drive) const;
    bool    isDriveMounted(int drive) const;

    // Takt (für DMA-/BUSRQ-Timing)
    void    clockTick();

private:
    A5120Config   cfg_;
    Z80PIO        ctrl_pio_{"K5122-CTRL-PIO"};   // Ports 10H–13H
    Z80PIO        data_pio_{"K5122-DATA-PIO"};   // Ports 14H–17H
    FloppyDrive*  drives_[4]  = {};
    int           active_drv_ = -1;      // Aktuell gewähltes Laufwerk
    int           current_trk_[4] = {};  // Aktuelle Spur pro Laufwerk
    bool          head_loaded_ = false;

    // Interne Kommando-State-Machine
    enum class FDCState {
        IDLE, STEPPING, HEAD_LOAD, READING, WRITING, SEEKING_T0
    };
    FDCState      state_ = FDCState::IDLE;
    int           step_count_ = 0;
    int           sector_pos_ = 0;      // Position im Sektor (beim Lesen)

    // PIO Callbacks
    void    onCtrlPortAWrite(uint8_t data);  // Steuersignale
    void    onCtrlPortBChange(uint8_t data); // /HF, PRE
    void    onDataPortAWrite(uint8_t data);  // Schreibdaten-Byte
    uint8_t onDataPortBRead();               // Lesedaten-Byte

    // Laufwerk-Operationen (intern, Image-basiert)
    void    startRead();
    void    startWrite();
    void    doStep(bool inward);
    void    seekTrack0();
    uint8_t readNextByte();
    void    writeNextByte(uint8_t data);

    // BUSRQ/WAIT Steuerung
    void    requestBus();    // /BUSRQ setzen
    void    releaseBus();    // /BUSRQ freigeben
    void    assertWait();    // /WAIT setzen
    void    releaseWait();
};
```

## 7. Floppy-Emulations-State-Machine

### 7.1 SEEK-Operation (Schritt-Befehle)

```
CPU schreibt Steuer-PIO Tor A:
  /ST = 0, MR/SD = 0/1 → ein Schritt
  → K5122::onCtrlPortAWrite()
  → active_drv_.seekStep(direction)
  → aktuelle_spur ± 1
  → Status: /TO = (spur == 0) ? 0 : 1
```

### 7.2 LESE-Operation (mit BUSRQ)

```
OS: /HL=0 (Kopf aufsetzen), /STR=0 (Start)
→ K5122 weiß: Sektor lesen
→ requestBus() → /BUSRQ=0 → ZVE2 übernimmt Bus
→ Für jeden Sektor-Byte:
    Daten-PIO Tor B = nächstes Byte aus Image
    PIO-Interrupt → ZVE2 liest Port 16H (Daten-PIO B)
    ZVE2 speichert in RAM (DMA-Transfer)
→ Nach komplettem Sektor: releaseBus()
```

### 7.3 Vereinfachter Emulations-Ansatz

Da die eigentliche PIO-Interrupt-Kommunikation zwischen K5122 und ZVE2 sehr komplex ist, kann der Emulator **abkürzend** vorgehen:

Wenn `/STR` gesetzt wird und der Sektor bekannt ist, transferiert der Emulator den **gesamten Sektor in einem Schritt** ohne einzelne Byte-PIO-Interrupts. Das OS wartet sowieso auf das Ende des Transfers.

```cpp
void K5122::onCtrlPortAWrite(uint8_t ctrl) {
    bool start = !(ctrl & (1 << 3));  // /STR = 0 → Kommando starten
    bool write_en = !(ctrl & (1 << 0));

    if (start && active_drv_ >= 0 && drives_[active_drv_]) {
        if (!write_en) {
            // Sektor lesen → direkt in RAM via K1520Bus schreiben
            auto& drv = *drives_[active_drv_];
            auto data = drv.readSector(current_trk_[active_drv_],
                                        head_, sector_);
            for (size_t i = 0; i < data.size(); i++)
                bus_.memWrite(dma_addr_ + i, data[i]);

            // Interrupt auslösen (Sektor fertig)
            ctrl_pio_.portBWrite(0x00);  // /RDYL = OK
            assertInterrupt();
        }
    }
}
```

## 8. Laufwerksauswahl (8212, Port 18H)

```
Bit-Belegung Port 18H (Schreibzugriff):
  DO1 = /LCK2   DO5 = /LCK3
  DO2 = /LCK1   DO6 = /LCK4
  DO3 = /SE3    DO7 = /SET
  DO4 = /SE4    DO8 = /SEZ

Laufwerksauswahl durch:
  /LCK1 = 0 → Laufwerk A (drive 0)
  /LCK2 = 0 → Laufwerk B (drive 1)
  /LCK3 = 0 → Laufwerk C (drive 2)
  /LCK4 = 0 → Laufwerk D (drive 3)
```
