# Feinentwurf: K5122 AFS – Floppy-Anschlußsteuerung

**Modul:** `core/cards/k5122/`  
**Dateien:** `k5122.h`, `k5122.cpp`  
**Dokumentation:** `doc/trascripted/Floppy Anschlußsteuerung K 5122.md`

---

## 1. Aufgabe

Die K5122 steuert bis zu 4 Diskettenlaufwerke. Sie ist keine direkte FDC-Implementierung wie der µPD765, sondern eine **PIO-basierte Lösung**: Zwei Z80-PIO-Chips übernehmen die Bit-by-Bit-Kommunikation mit den Laufwerken unter Steuerung der CPU.

Im Emulator wird die analoge Diskettenkommunikation durch **direkten Image-Zugriff** ersetzt.

---

## 2. Hardware-Zusammensetzung

```
K5122-Karte
├── Steuer-PIO A1.2 (Z80-PIO) – Ports 10H–13H
│   ├── Tor A Output: Steuersignale (/WE, MK, /STR, /ST, ...)
│   └── Tor B BitMode: Status-Ein-/Ausgänge (/RDYL, /TO, /WP, ...)
├── Daten-PIO A1.1 (Z80-PIO) – Ports 14H–17H
│   ├── Tor A Output: Schreibdaten (8 Bit)
│   └── Tor B Input: Lesedaten (8 Bit)
└── Laufwerksauswahl A4 (8212) – Port 18H
    └── /LCK1-4, /SE3, /SE4, /SET, /SEZ
```

## 3. I/O-Port-Belegung

| Port | PIO | Tor | Typ | Funktion |
|------|-----|-----|-----|----------|
| 10H | S | A | Data | Steuerausgänge |
| 11H | S | A | Ctrl | PIO-Steuerregister |
| 12H | S | B | Data | Status-Eingang/Ausgang |
| 13H | S | B | Ctrl | PIO-Steuerregister |
| 14H | D | A | Data | Schreibdaten (8 Bit) |
| 15H | D | A | Ctrl | PIO-Steuerregister |
| 16H | D | B | Data | Lesedaten (8 Bit) |
| 17H | D | B | Ctrl | PIO-Steuerregister |
| 18H | — | — | — | Laufwerksauswahl (8212) |

---

## 4. Steuersignale (Steuer-PIO Tor A Ausgänge)

| Bit | Signal | Funktion |
|-----|--------|----------|
| 0 | /WE | Write Enable (Schreibkopf aktivieren) |
| 1 | MK | Mark (Startmarkierung) |
| 2 | /FR | Fault Reset (Fehler rücksetzen) |
| 3 | /STR | Start (Kommando starten) |
| 4 | MK1 | Mark 1 |
| 5 | MR/SD | Mark Reset / Step Direction (0=inward, 1=outward) |
| 6 | /HL | Head Load (Schreib-/Lesekopf aufsetzen) |
| 7 | /ST | Step (ein Schritt in Richtung MR/SD) |

## 5. Status-Signale (Steuer-PIO Tor B Eingänge/Ausgänge)

| Bit | Signal | Richtung | Funktion |
|-----|--------|----------|----------|
| 0 | /RDYL | Eingang | Laufwerk bereit (MFS); 0 = bereit |
| 1 | MKE | Eingang | Marke erkannt |
| 2 | /HF | Ausgang | High Frequency (0=MFM 8", 1=FM 5") |
| 3 | PRE | Ausgang | Precompensation |
| 4 | /FA | Eingang | Fault Adapter |
| 5 | /WP | Eingang | Write Protect; 0 = schreibgeschützt |
| 6 | /FW | Eingang | **Fault** (Laufwerksfehler); 1 = kein Fehler |
| 7 | /TO | Eingang | **/TRACK 00** (Spur 00); 0 = Kopf auf Spur 0 |

> ⚠️ **Achtung – Bit 6/7 nicht verwechseln** (Quelle: `doc/trascripted/Floppy
> Anschlußsteuerung K 5122.md`, B₆=/FAULT=/FW, B₇=/TRACK 00=/TO). Das Boot-ROM
> wertet **Bit 7** (/TO = Spur 00) in der Laufwerkserkennung aus (ROM 0x0110:
> `IN (12H); RLCA; JR NC` → fortfahren, wenn Bit 7 = 0). `updateStatusPortB()`
> löscht daher Bit 7 bei Spur 0; Bit 6 (/FW) bleibt 1 (kein Fehler emuliert).

    // DMA-Protokoll (BUSRQ/BUSAK)
    // dmaUpdate(): Fallback für die ZVE1-Byte-Polling-Phase (ZVE2 im Reset/WAIT).
    //   Führt einen ausstehenden Schreib-Transfer aus und gibt BUSRQ frei.
    void dmaUpdate();
    // endDmaTransfer(): von A5120Machine aufgerufen, sobald ZVE2 die DMA durch

    void endDmaTransfer();

    // Direktzugriff (für Tests / C-API)
    FloppyDrive& drive(int idx) { return drives_[idx]; }

    void handleCtrlPortAWrite(uint8_t data);  // /ST, /STR Flanken-Erkennung
    void handleDataPortAWrite(uint8_t data);  // Byte in Schreib-Buffer

    FloppyDrive drives_[4];

    int     selected_drive_ = 0;

    size_t               buf_pos_      = 0;
    bool                 transferring_ = false;
    bool                 write_mode_   = false;

    // DMA-Zustandsmaschine
    bool dma_pending_  = false;               // true: BUSRQ assertiert, Transfer ausstehend
    bool dma_is_write_ = false;              // true = Schreiben, false = Lesen

    std::array<std::chrono::steady_clock::time_point, 4> led_until_{};

## 7. Transfer-Modell: ZVE2-DMA (BUSRQ/BUSAK)

Die K5122 überträgt Sektordaten über das BUSRQ/BUSAK-Protokoll. Beim realen
A5120-Boot führt **ZVE2** (zweite Z80-CPU auf der K2526) den Transfer aus; das
Boot-ROM startet ZVE2 aktiv (OUT 04H) und lässt es das STROBE_LOOP-Programm laufen.
Der frühere „Boot-ROM nutzt kein ZVE2"-Annahme ist überholt – Details und der
verifizierte Ablauf stehen in `doc/analyse_zre_rom_boot.md`.

### 7.1 Kontinuierlicher Track-Stream (`doReadSector()`)

Reale Hardware streamt die rotierende Spur byteweise; ZVE2 sucht im Strom nach der
IDAM-Adressmarke jedes Sektors und kopiert die Daten. Der Emulator bildet das nach,
indem `doReadSector()` bei einer Lese-/STR-Flanke die **gesamte (cyl,head)-Spur** als
Folge von **138-Byte-Sektorblöcken** in `sector_buf_` aufbaut:

```
Offset  Inhalt                       gelesen vom Boot-ROM bei
0       00   sync/gap                0x0261 (verworfen)
1       FE   IDAM-Adressmarke        0x020B (CP B' = 0xFE)
2       cyl  Zylinder                0x0210 (CP E')
3       head Kopf/Seite              0x0215 (CP D')
4       sec  Sektor-ID (1-basiert)   0x021A (CP L')
5       size Size-Code (128→0 …)     0x021F (CP H')
6..7    00   IDAM-CRC / Gap (2 B)    0x022F, 0x0239 (verworfen)
8..135  data Nutzdaten (128 B)       0x023C…0x0242 (INI/INIR)
136..137 00  Daten-CRC (2 B)         0x0245, 0x0247 (nachgelagerte INI)
```

**Die Blockgröße 138 ist exakt** = wie viele Port-16H-Lesezugriffe das Boot-ROM pro
Sektor macht (6 Header + 2 IDAM-CRC + 128 Daten + 2 Daten-CRC). Ein zu kurzer Block
verschiebt den Strom pro Sektor und die IDAM-Suche scheitert ab Sektor 2.
`ioRead(0x16)` streamt `sector_buf_` und springt am Ende auf 0 zurück (die Diskette
dreht weiter); es gibt **kein** BUSRQ-Release beim Leeren des Puffers mehr.

### 7.2 LESE-DMA mit ZVE2

```
/STR fallende Flanke (Port 10H Bit 3) mit /WE=1 – ZVE1-Kontext:
  K5122::handleCtrlPortAWrite()
    → doReadSector()        ← ganze Spur als 138-Byte-Block-Stream laden
    → bus_.assertBUSRQ()    ← DMA-Anforderung

A5120Machine::run():
  if (bus_.isBUSRQ() && ZVE2 aktiv) {
      zre_.zve2Step();                              // ZVE2 sucht IDAMs, INIR
      if (bus_.memRead(0x03F8) == 3)                // ZVE2-Completion?
          afs_.endDmaTransfer();                    // → BUSRQ freigeben
      // KEIN continue: ZVE1 wird ebenfalls gesteppt (Nebenläufigkeit, s.u.)
  }

ZVE2 (STROBE_LOOP, ROM 0x01DD): sucht je Sektor die IDAM, vergleicht
  cyl/head/sec/size, überträgt 128 Datenbytes per INI/INIR, bis sector_id==[07F2].
  Abschluss (ROM 0x0267): OUT(11H,03); LD ([03F8H]),3.
```

**BUSRQ-Freigabe erst bei Completion**, nicht beim Puffer-Leeren: ZVE2 meldet das
Ende durch `[0x03F8]=3`; `A5120Machine` erkennt das und ruft `endDmaTransfer()`.

**Nebenläufiges ZVE1/ZVE2-Stepping:** Auf echter Hardware laufen beide CPUs parallel
(/BUSRQ taktet nur einzelne Buszyklen). Der Run-Loop steppt deshalb während der DMA
ZVE2 **und** ZVE1. Reihenfolge entscheidend: ZVE1 muss `CALL 0194` beenden (dessen
Ende `[0x03F8]=0` bei 0x01B3 schreibt) und in die Poll-Schleife 0x0168 gelangen,
**bevor** ZVE2 `=3` schreibt – sonst überschreibt ZVE1s spätes `=0` das `=3`.

### 7.3 SCHREIB-DMA mit ZVE2

```
/STR fallende Flanke mit /WE=0 – ZVE1-Kontext:
  handleCtrlPortAWrite() → sector_buf_.clear(); bus_.assertBUSRQ()

ZVE2-DMA-Programm: OTIR (Bytes (HL) → Port 14H), dann erneuter /STR-Puls mit /WE=0.

handleCtrlPortAWrite() – /STR-Flanke bei gehaltenem BUSRQ + /WE=0:
  doWriteSector()  ← sector_buf_ ins Disk-Image schreiben
  bus_.releaseBUSRQ()
```

### 7.4 Fallback: ZVE1-Byte-Polling (ZVE2 im Reset/WAIT)

Wenn ZVE2 im Reset oder durch /WAIT-ZVE2 angehalten ist (`isZVE2InReset()` bzw.
`isZVE2Waiting()`), übernimmt `dmaUpdate()` die Bus-Freigabe, und ZVE1 liest den
gefüllten `sector_buf_` selbst byteweise per `IN A,(16H)`. Diese Pfad wird im realen
A5120-Boot **nicht** benutzt (dort läuft ZVE2), bleibt aber für OS-Szenarien und
Tests erhalten.

### 7.5 Laufwerksstatus (ctrl_pio_ Port B)

Nach jeder Statusänderung schreibt `updateStatusPortB()` das Statusbyte in den
Eingangslatch von ctrl_pio_; die CPU liest es über Port 0x12.

```
Kein Laufwerk: 0xF5 = /TO=1, /FW=1, /WP=1, /FA=1, /HF=1, PRE=0, MKE=0, /RDYL=1
Laufwerk eingelegt:  /RDYL=0 (bereit, Bit 0)
Auf Spur 0:          /TO=0   (Bit 7 = /TRACK 00; vom Boot-Laufwerkstest geprüft)
Schreibgeschützt:    /WP=0   (Bit 5)
/FW (Bit 6, Fault) bleibt 1: es wird kein Laufwerksfehler emuliert.
```

## 8. Laufwerksauswahl (8212, Port 18H)

```
Bit-Belegung Port 18H (Schreibzugriff):
  DO1 = /LCK2   DO5 = /LCK3
  DO2 = /LCK1   DO6 = /LCK4
  DO3 = /SE3    DO7 = /SET
  DO4 = /SE4    DO8 = /SEZ

Laufwerksauswahl durch:
  /LCK1 = 0 → Laufwerk A (drive 0)
  /LCK2 = 0 → Laufwerk B (drive 1)
  /LCK3 = 0 → Laufwerk C (drive 2)
  /LCK4 = 0 → Laufwerk D (drive 3)
```
