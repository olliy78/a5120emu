## 1. Aufgabe

Der Bus-Simulator implementiert das K1520-Systembus-Protokoll (TGL 37271/01) als Software-Abstraktion. Er ist die zentrale Kommunikationsschicht zwischen allen emulierten Komponenten.

## 2. K1520Bus-Klasse

### 2.1 Geräteregistrierung

```cpp
class BusDevice {
public:
    virtual ~BusDevice() = default;
    virtual uint8_t ioRead(uint8_t port)            { return 0xFF; }
    virtual void    ioWrite(uint8_t port, uint8_t d) {}
    virtual const char* deviceName() const           { return "?"; }
};

class MemDevice {
public:
    virtual ~MemDevice() = default;
    virtual uint8_t  memRead(uint16_t addr)             = 0;
    virtual void     memWrite(uint16_t addr, uint8_t d) = 0;
    virtual bool     isWritable() const { return true; }  // false = ROM
};

class InterruptSlave {
public:
    virtual ~InterruptSlave() = default;
    virtual void setIEI(bool iei)   = 0;   // IEI-Eingang gesetzt/gelöscht
    virtual bool getIEO() const     = 0;   // IEO-Ausgang lesen
    virtual bool hasInterrupt() const = 0; // Wartet auf Bestätigung?
    virtual uint8_t getVector() const { return 0xFF; } // Interrupt-Vektor
};
```

### 2.2 K1520Bus-Klasse vollständig

```cpp
class K1520Bus {
public:
    K1520Bus() = default;
    ~K1520Bus() = default;

    // ─── Geräteregistrierung ─────────────────────────────────────

    // I/O-Gerät: basePort..basePort+numPorts-1 werden an dev geleitet
    void registerIO(BusDevice* dev, uint8_t basePort, uint8_t numPorts = 1);

    // Speichergerät: base..base+size-1 werden an dev geleitet
    void registerMem(MemDevice* dev, uint16_t base, uint16_t size);

    // Interrupt-Prioritätskette (Reihenfolge = absteig. Priorität)
    // Erster Eintrag hat höchste Priorität
    void setInterruptChain(std::initializer_list<InterruptSlave*> chain);

    // ─── Busoperationen (von Z80CPU aufgerufen) ──────────────────

    uint8_t memRead(uint16_t addr);
    void    memWrite(uint16_t addr, uint8_t data);
    uint8_t ioRead(uint8_t port);
    void    ioWrite(uint8_t port, uint8_t data);

    // Interrupt-Acknowledge-Zyklus: Z80 bestätigt INT
    // Rückgabe: Interrupt-Vektor (Modus 2) oder 0xFF
    uint8_t interruptAcknowledge();

    // ─── Signale (von Karten gesetzt) ────────────────────────────

    void assertINT();            // /INT anfordern (beliebige Karte)
    void releaseINT();           // /INT freigeben
    void assertNMI();            // /NMI (einmalig)
    void assertRESET();          // /RESET senden
    void assertWAIT();           // /WAIT (Buszyklen verlängern)
    void releaseWAIT();

    // /MEMDI: Speicherzugriff global sperren (BS-PIO)
    void setMEMDI(bool disabled);
    bool getMEMDI() const;

    // /IODI: I/O-Zugriff global sperren
    void setIODI(bool disabled);

    // ─── Zustandsabfrage ─────────────────────────────────────────

    bool isINT()    const;
    bool isNMI()    const;
    bool isWAIT()   const;
    bool isRESET()  const;
    bool isHALT()   const;
    bool isBUSRQ()  const;

    // ─── Listener (für Logging/Debugging) ────────────────────────
    using BusTrace = std::function<void(bool isIO, bool isRead,
                                        uint16_t addr, uint8_t data)>;
    void setTraceCallback(BusTrace cb);

private:
    // I/O-Dispatch-Tabelle: port → BusDevice*
    std::array<BusDevice*, 256> io_map_{};

    // Speicher-Dispatch: sorted list (base, size, device)
    struct MemRegion { uint16_t base, size; MemDevice* dev; };
    std::vector<MemRegion> mem_regions_;

    // Interrupt-Kette
    std::vector<InterruptSlave*> int_chain_;

    // Signalzustand
    bool int_asserted_  = false;
    bool nmi_pending_   = false;
    bool wait_asserted_ = false;
    bool memdi_         = false;
    bool iodi_          = false;

    BusTrace trace_cb_;

    // Interne Hilfsfunktionen
    MemDevice* findMemDevice(uint16_t addr) const;
    void       updateInterruptChain();
};
```

### 2.3 Speicher-Dispatch

Überlappende Speicherbereiche sind erlaubt (z.B. Lade-ROM überdeckt teilweise RAM). Der zuletzt registrierte Bereich hat Vorrang (wie echte Logik mit /RDY-Signal).

```cpp
uint8_t K1520Bus::memRead(uint16_t addr) {
    if (memdi_) return 0xFF;
    // Durchsuche alle registrierten Bereiche, letzter Treffer gewinnt
    MemDevice* hit = nullptr;
    for (auto& r : mem_regions_)
        if (addr >= r.base && addr < r.base + r.size)
            hit = r.dev;
    if (hit) return hit->memRead(addr);
    return 0xFF;  // Bus-Float
}
```

### 2.4 Interrupt-Daisy-Chain

Die Daisy-Chain verwaltet IEI/IEO gemäß Z80-Protokoll:

1. IEI des ersten Geräts ist immer aktiv (= High)
2. Gerät mit aktivem IEI und eigenem Interrupt-Wunsch setzt IEO = Low → alle nachfolgenden Geräte werden blockiert
3. Beim Interrupt-Acknowledge sendet das Gerät mit aktivem IEI seinen Vektor
4. Nach RETI gibt das Gerät IEO wieder frei

```cpp
void K1520Bus::updateInterruptChain() {
    bool iei = true;
    for (auto* dev : int_chain_) {
        dev->setIEI(iei);
        if (iei && dev->hasInterrupt())
            iei = false;  // blockiert alle nachfolgenden
        // IEO des aktuellen = IEI des nächsten (wenn kein eigener INT)
        iei = iei && dev->getIEO();
    }
}

uint8_t K1520Bus::interruptAcknowledge() {
    for (auto* dev : int_chain_) {
        if (dev->hasInterrupt())
            return dev->getVector();
    }
    return 0xFF;
}
```

## 3. Koppelbus-Klasse

Der Koppelbus modelliert die Wickelbrücken-Verdrahtung. Er ist kein Protokoll-Bus, sondern ein Signal-Router mit benannten Leitungen.

### 3.1 Koppelbus-Signale (A5120)

| Signal | Richtung | Beschreibung |
|--------|----------|--------------|
| `/IEI1`, `/IEO1` | Kette | Zweite Interrupt-Prioritätskette |
| `ZC_TO0`–`ZC_TO2` | Output K2526 CTC | Zero-Count/Timer-Ausgänge |
| `CLK_TRG0`–`CLK_TRG3` | Input CTC Kanäle | Externe Taktquellen |
| `/SUE` | Output K2526 | Spannungsüberwachung |
| `/MEMDI1`, `/MEMDI2` | Output K2526 | Speicherbereichsumschaltung |
| `/SA` | Output K2526 | Sonderausgang (Netzaus) |

### 3.2 Implementierung

```cpp
class KoppelbusSignal {
public:
    using Callback = std::function<void(bool level)>;

    void connect(Callback receiver) { receivers_.push_back(receiver); }
    void drive(bool level) {
        if (level != current_) {
            current_ = level;
            for (auto& cb : receivers_) cb(level);
        }
    }
    bool read() const { return current_; }

private:
    bool current_ = true;  // Default: nicht aktiv (positive Logik)
    std::vector<Callback> receivers_;
};

class Koppelbus {
public:
    // Benannte Signalleitungen
    KoppelbusSignal iei1, ieo1;    // Zweite Interrupt-Kette
    KoppelbusSignal zc_to[3];      // CTC Zero-Count-Ausgänge
    KoppelbusSignal clk_trg[4];    // CTC Takt-Eingänge
    KoppelbusSignal sue;           // Spannungsüberwachung
    KoppelbusSignal memdi1, memdi2;
    KoppelbusSignal sa;            // Netzausschaltung

    // Generische Methode für maschinen-spezifische Signale
    KoppelbusSignal& signal(const std::string& name);

private:
    std::unordered_map<std::string, KoppelbusSignal> extra_signals_;
};
```

## 4. Fehlerfälle und Bus-Float

- **Kein Gerät antwortet:** `memRead` / `ioRead` gibt `0xFF` zurück (Bus-Float)
- **Überlappende I/O-Bereiche:** `registerIO` wirft Exception (Konfigurationsfehler)
- **Schreibzugriff auf ROM:** `MemDevice::isWritable() == false` → wird ignoriert

class Z80CPU {
public:
