# Feinentwurf: K3526 OPS – Operationsspeicher

**Modul:** `core/cards/k3526/`  
**Dateien:** `k3526.h`, `k3526.cpp`  
**Dokumentation:** `doc/trascripted/Speichersteckeinheiten K 3520, 3521, 3525, 3820.md`

---

## 1. Aufgabe

Die K3526 ist die Arbeitsspeicherkarte des A5120. Sie enthält 64 KB DRAM, aufgeteilt in **4 Gruppen à 16 KB**. Jede Gruppe ist über DIL-Schalter unabhängig auf einen beliebigen 16KB-ausgerichteten Adressbereich konfigurierbar. Die Karte hat kein I/O-Interface; Banking und Schreibschutz erfolgen über `/MEMDI` auf dem K1520-Bus (gesteuert von K2526).

---

## 2. Hardware-Konfiguration

### 2.1 Adressgruppen (DIL-Schalter)

Jede der 4 Gruppen (I–IV) hat 16KB RAM und wird durch drei DIL-Schalter konfiguriert:

| Schalter | Bit | Bedeutung |
|----------|-----|-----------|
| A5 | Adressbit 15 (64KB/32KB-Grenze) | 0=unten (0000H–7FFFH), 1=oben |
| A7 | Adressbit 14 | selektiert 32KB-Block |
| A10 | Adressbit 13 | selektiert 16KB-Block innerhalb des 32KB-Blocks |

Konfiguration für den A5120-Standard (64KB RAM, vollständiger Adressraum):

| Gruppe | A5 | A7 | A10 | Adressbereich |
|--------|----|----|-----|--------------|
| I      | 0  | 0  | 0   | 0000H–3FFFH  |
| II     | 0  | 1  | 0   | 4000H–7FFFH  |
| III    | 1  | 0  | 0   | 8000H–BFFFH  |
| IV     | 1  | 1  | 0   | C000H–FFFFH  |

### 2.2 MEMDI-Auswahl pro Gruppe

Jede Gruppe hat einen eigenen DIL-Schalter für die `/MEMDI`-Quelle:

| MEMDI-Schalter | Quelle |
|----------------|--------|
| MEMDI          | `/MEMDI` des K1520-Systembus |
| MEMDI1         | `/MEMDI1` des Koppelbusses |
| MEMDI2         | `/MEMDI2` des Koppelbusses |

Beim A5120 nutzen alle Gruppen den normalen `/MEMDI`-Eingang.

### 2.3 RFSH-Varianten

| Variante | Refresh |
|----------|---------|
| K3526.00/.20/.40 | Interne Refresh-Logik (kein /RFSH vom Bus benötigt) |
| K3526.10/.30/.50 | Externes /RFSH-Signal vom K1520-Bus |

Im Emulator wird kein Refresh emuliert (kein funktionaler Unterschied).

## 3. Speicher-Map (A5120-Standardkonfiguration)

```
0000H–3FFFH:  Gruppe I  (nach Boot-ROM-Abschaltung vollständig RAM)
4000H–7FFFH:  Gruppe II
8000H–BFFFH:  Gruppe III
C000H–FFFFH:  Gruppe IV (F800H–FFFFH überdeckt von K7024 VRAM)
```

Priorisierung durch Reihenfolge der `registerMem()`-Aufrufe am K1520Bus:
1. K3526 wird zuerst registriert (alle 4 Gruppen)
2. K7024 VRAM danach → überdeckt F800H–FFFFH
3. K2526 Lade-ROM zuletzt → überdeckt 0000H–03FFH nach /RESET

## 4. Klassen-Interface

```cpp
struct K3526GroupConfig {
    uint16_t base_addr;      // 16KB-ausgerichtet: 0x0000, 0x4000, 0x8000, 0xC000
    bool     memdi_source;   // false=MEMDI(Bus), true=MEMDI1/MEMDI2(Koppelbus)
    // (Koppelbus-Quelle als separate Verbindung verdrahtet)
};

class K3526 : public MemDevice {
public:
    struct A5120Config {
        // Standard: 4 Gruppen à 16KB, vollständiger 64KB-Raum
        K3526GroupConfig groups[4] = {
            {0x0000, false}, {0x4000, false},
            {0x8000, false}, {0xC000, false}
        };
    };

    explicit K3526(const A5120Config& cfg = {});

    // MemDevice Interface (wird für jede Gruppe separat registriert)
    uint8_t memRead(uint16_t addr) override;
    void    memWrite(uint16_t addr, uint8_t data) override;
    bool    isWritable() const override { return true; }

    // Registrierung am K1520Bus (jede Gruppe als eigenes MemDevice)
    void attachToBus(K1520Bus& bus);

    // MEMDI-Eingang per Gruppe (für Koppelbus-Verdrahtung)
    void setMemDI(int group, bool disabled);

    // Direktzugriff (für Tests und Initialzustand)
    void    fill(uint8_t value = 0x00);
    void    load(uint16_t addr, const uint8_t* data, uint16_t len);
    const uint8_t* rawPtr() const;

private:
    A5120Config cfg_;
    std::array<uint8_t, 65536> mem_{};
    bool group_memdi_[4] = {};  // true = Gruppe gesperrt
};
```

---

## 5. DRAM-Refresh

Echter DRAM benötigt periodische Refresh-Zyklen. Im Emulator wird DRAM-Refresh **nicht emuliert** – kein funktionaler Unterschied, da keine Software auf Refresh-Ausfall angewiesen ist.

## 6. Testbarkeit

```python
# tests/python/test_k3526.py
def test_read_write(k3526_lib):
    ops = k3526_lib.k3526_create()
    k3526_lib.k3526_write(ops, 0x1234, 0xAB)
    assert k3526_lib.k3526_read(ops, 0x1234) == 0xAB

def test_group_memdi_blocks_access(k3526_lib):
    ops = k3526_lib.k3526_create()
    k3526_lib.k3526_write(ops, 0x4000, 0xCC)  # Gruppe II
    k3526_lib.k3526_set_memdi(ops, 1, True)   # Gruppe II sperren
    assert k3526_lib.k3526_read(ops, 0x4000) == 0xFF  # Bus-Float

def test_initial_state(k3526_lib):
    ops = k3526_lib.k3526_create()
    k3526_lib.k3526_fill(ops, 0x00)
    assert k3526_lib.k3526_read(ops, 0x5000) == 0x00
```
